The Internet

It is the internet that makes the WWW possible. The internet is a global system of connected devices, routers, and modems.

Image source: http://www.theshulers.com/whitepapers/internet_whitepaper/index.html

People will often think of the internet and the World Wide Web (aka WWW) interchangeable but the internet is specifically the layers of networking that exist to make the connection of devices possible. Most people are connected to this network by paying an Internet Service Provider (aka ISP) for access in the form of a wired connect directly to a business/home or a wireless connection from a cellular tower or satellite.

No one person, company, or organization or government runs the internet.

Image source: http://www.bitrebels.com/technology/find-out-who-runs-the-internet-chart/

TCP/IP

TCP/IP is roughly the baseline protocol that is used so that interconnect devices on the internet can communicate with each other. In other words the internet infrastructure connects devices to each other, but TCP/IP is the baseline language/interface (i.e protocols) that allow devices (computers, laptops, phones etc..) to speak to each other in a common way (i.e. TCP/IP defines how data travels on the internet).

The Transmission Control Protocol (aka TCP) specifically directs packets (i.e. chunked data) of information to a specific application on a computer connected to the network that can be identified using a Internet Protocol Address (aka IP address).

The Web Server and The Web Browser/Client

The internet and TCP/IP made it possible for one device to exchange data with another device on the network using basically ones and zeros (i.e. packets of data). On the world wide web when dealing with web pages a specific type of data exchange is taking place. Typically a web browser or client/user-agent is asking for data from a specific location/device serving a very specific type of data over the network (e.g. a computer running server software on the network/internet). This requires a specific protocol sitting on top of the baseline TCP/IP protocols. Basically, the WWW runs because a client can connect to a web server through the internet, and this client can talk to the web server use the Hypertext Transfer Protocol (aka HTTP) to exchange text data most commonly written in the Hyper Text Markup Language (aka HTML).

HTTP

Hypertext Transfer Protocol (aka HTTP) is the foundation protocol that sits on top of the TCP/IP protocols. In terms of the WWW HTTP is the foundational system that allows a user to request a specific document/file (i.e. request .html, .jpg, .js, .css etc...) on the internet via a Universal Resource Locator (aka URL) and for the web server to give a response (i.e. send the .html, .jpg, .js, .css files to the client).

HTML

When a user enters a URL into a web browser/client, an HTTP request is sent over the internet to a web server typically asking the server to send back a specific an .html document/file containing HTML. What is returned is a document containing information written in the Hyper Text Markup Language that is then used by the web browser/client to visually display information to the user. It might be helpful to think of an HTML document as a very particular kind of text document that contains meta data, text, images, and videos wrap with semantical meaning to be displayed to the user.
